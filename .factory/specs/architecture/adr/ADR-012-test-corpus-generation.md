# ADR-012: Test corpus generation strategy

## Context

BC-2.16.006 requires a reproducible synthetic corpus for scale testing. NFR-004, NFR-005, NFR-006, and NFR-007 all require measurement against a controlled corpus. The corpus must be:
- Reproducible (given the same seed, generates the same corpus)
- Parameterizable (size in sources, word count distribution, topic distribution)
- Fast to generate (must not add hours to the CI run)
- Valid (generated sources must pass hook validation — valid frontmatter, valid wikilinks, valid filenames)

## Decision

### Script interface

`scripts/gen-test-corpus.sh` — a bash script (consistent with ADR-001 bash-only stack).

```
Usage: scripts/gen-test-corpus.sh [OPTIONS] <output-dir>

Options:
  --sources N       Number of source files to generate (default: 100)
  --seed N          Deterministic seed for random generation (default: 42)
  --topics LIST     Comma-separated topic categories (default: ai,health,psychology,productivity,business,books,podcasts)
  --avg-words N     Average word count per source (default: 3000)
  --wiki-ratio N    Wiki pages generated per source (default: 5)
  --format FORMAT   Output format: brain-vault (default) | json-manifest-only

Exit codes: 0 success; 1 advisory (some files skipped); 2 generation failed
```

### Output format

When `--format brain-vault` (default), the script generates a complete brain vault tree:
```
<output-dir>/
  sources/
    ai/synthetic-source-0001.md
    health/synthetic-source-0002.md
    ...
  wiki/
    concepts/synthetic-concept-0001.md
    ...
  wiki/index.md                        (index of all generated pages)
  wiki/log.md                          (log of all generation events)
  .brain/
    manifest.json                      (pre-populated with all sources)
    logs/ingest-tokens.jsonl           (synthetic token records)
```

Each synthetic source file has valid frontmatter:
```markdown
---
title: "Synthetic Source 0001"
url: "https://synthetic.example.com/source-0001"
ingested_at: "2026-01-01T00:00:00Z"
topic: ai
---
# Synthetic Source 0001

[Generated content: N words of lorem-ipsum-style text with wikilinks to generated concepts]
```

Each synthetic wiki page has valid frontmatter including `embedding_status: pending`.

Wikilinks in synthetic sources point to synthetic wiki pages that exist in the corpus — validated cross-references, not dangling links.

### Deterministic generation

The seed controls `shuf --random-source=<(seq $SEED)` for word and topic distribution. Given the same seed and parameters, the script produces byte-identical output. This is verifiable by running the script twice and diffing the output directories.

### Integration with scale-test workflow

`workflows/scale-test.yaml` calls `gen-test-corpus.sh --sources 10000` as its first step, then pipes the output directory to the ingest pipeline for timing measurement. The workflow captures timing and memory metrics per step and writes them to `.brain/logs/scale-test-YYYY-MM-DD.jsonl`.

### Fixture corpus for bats tests

A smaller fixture corpus (`plugins/brain-factory/tests/fixtures/smoke-brain/`) is NOT generated by this script — it is hand-authored and version-controlled. The smoke-brain fixture covers positive + negative + edge cases for each hook. `gen-test-corpus.sh` is for scale testing only; it does not replace hand-authored fixtures.

## Consequences

**Positive:**
- Reproducible corpus means scale test results are comparable across runs and commits
- Bash-only implementation keeps the build dependency surface minimal
- The script is a Phase 3 deliverable; its interface is specced now so Phase 2 stories can reference it correctly

**Negative:**
- Synthetic content does not capture the statistical properties of real-world articles (word count distribution, topic clustering). The scale test validates architectural scale properties, not real-world quality metrics.
- Generating 10K sources at ~3K words each produces ~30M words of synthetic text — this takes non-trivial time. Expected: ~10 minutes on GitHub Actions. Scale test is run on-demand, not in the default CI matrix.

**Neutral:**
- The `--wiki-ratio` parameter produces synthetic wiki pages that pass frontmatter validation (VP-005) by construction — not a substitute for end-to-end wiki generation quality testing
