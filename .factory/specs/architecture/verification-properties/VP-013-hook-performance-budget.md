# VP-013: Hook p99 latency under 100ms

## Property Statement

For every hook script, the wall-clock time from hook invocation (stdin delivered) to exit (verdict written to stdout) is under 100ms at the p99 percentile when measured with a canonical 2KB payload on GitHub Actions ubuntu-latest. This applies to all 13 hooks.

The sub-linear latency property for wikilink resolution (BC-2.02.007): the wikilink integrity hook's latency when processing a page with 5 wikilinks against a 10K-page index remains under 100ms p99 (O(n) grep-F is fast enough at 10K lines).

## Verification Mechanism

bats (hooks.bats) — timing assertion per hook:

```bash
@test "quarantine-fetch.sh: p99 latency < 100ms on canonical payload" {
  # Run 20 invocations (p99 = 19th percentile out of 20)
  local times=()
  for _ in {1..20}; do
    local start; start=$(date +%s%3N)
    echo '{"tool":"WebFetch","input":{"url":"https://paulgraham.com/think.html"}}' \
      | "${CLAUDE_PLUGIN_ROOT}/hooks/quarantine-fetch.sh" >/dev/null
    local end; end=$(date +%s%3N)
    times+=($((end - start)))
  done
  
  # Sort and take the 19th value (p99 of 20 samples)
  local sorted; sorted=($(printf '%s\n' "${times[@]}" | sort -n))
  local p99="${sorted[18]}"
  assert [ "$p99" -lt 100 ] "quarantine-fetch.sh p99 latency: ${p99}ms (exceeds 100ms)"
}
```

This test runs for each of the 13 hooks with representative canonical payloads. Note: the quarantine hook's curl pre-check adds ~30-50ms for a real URL fetch; it uses a localhost fixture URL in the test to avoid network dependency in the bats suite.

**Wikilink hook scale timing:**
```bash
@test "validate-wikilink-integrity.sh: < 100ms with 10K-page index" {
  local start; start=$(date +%s%3N)
  local payload='{"tool":"Write","input":{"path":"wiki/concepts/test.md","content":"See [[zettelkasten]] for more."},"output":{}}'
  WIKI_INDEX="${BATS_TEST_DIRNAME}/fixtures/wiki-index-10k.txt" \
    echo "$payload" | "${CLAUDE_PLUGIN_ROOT}/hooks/validate-wikilink-integrity.sh" >/dev/null
  local end; end=$(date +%s%3N)
  assert [ "$((end - start))" -lt 100 ]
}
```

## Assumed Prerequisites

- CI environment: GitHub Actions ubuntu-latest (NFR-001 measurement environment)
- `tests/fixtures/wiki-index-10k.txt` generated by gen-test-corpus.sh
- hooks that call external services (quarantine curl) use localhost fixtures in bats

## Counterexamples

- `validate-wikilink-integrity.sh` reads all wiki pages (O(n) pages × content length) instead of just the index — latency grows to seconds at 10K pages
- A hook that calls an external service with no timeout (can block indefinitely)
- bash startup overhead combined with jq parsing of a large stdin payload pushes p99 above 100ms
