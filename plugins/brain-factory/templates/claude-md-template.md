# Brain CLAUDE.md

This file guides Claude Code when working within this brain vault.

## Brain Structure

- `sources/` — immutable source files, organized by topic category
- `wiki/` — synthesized knowledge pages (concepts, people, frameworks, syntheses, observations, questions)
- `inbox/` — staging area for new items awaiting processing
- `briefs/` — generated briefs (daily, weekly, monthly, content, decisions, research)
- `rules/` — governance files (voice-avoid-list.txt)
- `.brain/` — brain metadata (manifest.json, STATE.md, policies.yaml, logs/)
- `.github/workflows/` — automation workflows

## Key Rules

1. Sources in `sources/` are immutable — never edit after ingest.
2. Wiki pages use `embedding_status: pending` until vector-indexed (v1.0+).
3. All wiki page frontmatter must include `type` and `embedding_status` fields.
4. Use `/brain:` skills for structured operations — do not manually reorganize the vault.

## Skills Available

- `/brain:init` — Initialize a new brain vault
- `/brain:ingest-url` — Ingest a web page into sources/
- `/brain:ingest-rss-opml` — Ingest RSS feeds
- `/brain:write-page` — Create or update a wiki page
- `/brain:search-brain` — Search across wiki and sources
- `/brain:synthesize-topic` — Generate synthesis wiki pages
- `/brain:chat-brain` — Conversational Q&A over the brain
- `/brain:monthly-synthesis` — Monthly synthesis brief
- `/brain:health` — Brain health check
